## Core thesis
A large-scale analysis of AI-generated pull requests reveals that agentic systems tackle performance optimizations across the full software stack, but their success depends heavily on when and how they intervene.

## Why it matters for Sapho
As AI agents assume greater responsibility for code generation, understanding their optimization behavior clarifies where they deliver value and where human oversight remains critical. These patterns inform how teams should deploy AI assistance and set expectations for review workflows.

## Key findings
- **Breadth across the stack:** AI agents address 52 distinct performance topics spanning ten high-level categories, indicating optimization capability at every software layer.
- **Timing matters:** Performance work by AI agents concentrates in the development phase, with noticeably less attention during maintenance, suggesting a gap in ongoing performance stewardship.
- **Acceptance varies by type:** The nature of an optimization significantly influences whether it gets merged and how long review takes, meaning not all performance improvements are treated equally.

## Limits
The analysis draws from abstract and introduction materials; the full experimental methodology, dataset scope, and detailed caveats lie beyond the excerpt reviewed.
