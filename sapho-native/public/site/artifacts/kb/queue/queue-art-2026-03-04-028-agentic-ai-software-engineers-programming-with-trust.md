## Thesis

Agentic software engineering only scales in production when trust mechanisms (verification, analysis, and accountable orchestration) are designed as first-class outputs rather than afterthoughts.

## Mechanism summary

This CACM opinion piece argues that high-trust deployment is the core bottleneck for AI software engineers, not raw code generation capability. The key mechanism is the move from single-shot LLM coding to agentic workflows that pair generation with analysis tooling so trust can be established at least at human-engineering parity. The implied shift is from programming at scale to programming with trust, where orchestration quality and verification coverage determine practical adoption.
