## Core thesis
Multi-agent systems for code generation exhibit a critical robustness gap between planning and execution components. This systematic study identifies how failures propagate across the planner-coder boundary and proposes targeted mitigation strategies to improve system reliability.

## Why it matters for Sapho
As organizations increasingly deploy multi-agent systems for software development tasks, unrecognized robustness flaws could compromise code quality, introduce security vulnerabilities, and erode trust in automated engineering pipelines. Understanding where and why these systems fail is essential before they assume greater responsibility in production environments.

## Key findings
- The "planner-coder gap" represents a distinct failure mode where coordination breakdowns between architectural planning and implementation agents degrade overall system performance
- Systematic empirical analysis reveals critical robustness vulnerabilities not captured by standard accuracy metrics
- Targeted mitigation strategies demonstrably bridge the gap and improve cross-agent coordination reliability
