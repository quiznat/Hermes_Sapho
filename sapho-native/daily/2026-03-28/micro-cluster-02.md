## Cluster Thesis

This cluster captures a maturation moment in LLM-based software engineering: the shift from ad-hoc prompting and experimental agents toward systematic, production-grade methodologies. The papers collectively argue that raw model capability is no longer the bottleneck—rather, the field now requires structured frameworks, verifiable workflows, and disciplined engineering practices to deploy multi-agent systems reliably at scale.

## Shared Signals

- The "crisis" of unstructured development practices that fail to meet production reliability standards
- Structured workflows and procedures as the primary solution pattern, replacing free-form interaction
- Multi-agent coordination requiring explicit architecture rather than emergent behavior
- Verification and testing stages as essential components, not afterthoughts

## Article Notes

- Promptware Engineering: Adaptation of classic software engineering principles to prompt-enabled systems, treating prompt development as a disciplined engineering discipline with lifecycle management
- A Survey on Code Generation with LLM-based Agents: Comprehensive taxonomy cataloging how agents now span the full development lifecycle—from design to maintenance—establishing a shared baseline for the field
- A New Context Engineering Workflow for Multi-Agent LLM Code Assistants: Demonstrated multi-tool ensemble approach for improving accuracy and reliability on production codebases through structured context management
- MetaGPT: Meta-programming framework replacing unstructured multi-agent chat with SOP-encoded assembly-line workflows and explicit verification stages
