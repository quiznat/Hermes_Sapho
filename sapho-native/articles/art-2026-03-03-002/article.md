# Agent READMEs: How Developers Keep AI Coders on Track

## Core Thesis

A new empirical study examines how developers craft context files to guide AI coding agents, revealing patterns in structure, maintenance practices, and instruction types—while exposing a notable gap in non-functional requirements specification.

## Why It Matters

As agentic coding tools become standard in software development, understanding how practitioners document intent and constraints for AI systems becomes critical for improving tool design, documentation standards, and the reliability of AI-assisted engineering workflows.

## Key Findings

- Developers maintain agent context files with varying frequency and structure, suggesting the practice is still evolving
- Instructions within these files can be systematically classified, pointing toward potential automation opportunities
- Non-functional requirements—such as performance constraints, security policies, and architectural standards—appear underspecified in current context files

This synthesis draws from an excerpt of the study's introductory sections and curator notes; the full experimental data, detailed methodology, and specific quantitative results referenced (including instruction type percentages) lie outside the available material.
