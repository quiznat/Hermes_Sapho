- NL2Repo-Bench evaluates coding agents on from-scratch Python library generation by giving them only a natural-language requirements document and an empty workspace, while withholding scaffolding, source code, and test cases during development and grading the final repository against the original upstream pytest suite.
- The released benchmark is a bounded but nontrivial Python-repository set, with 104 tasks across nine categories, long requirement documents, and eligibility filters that favor recent repositories with meaningful size, visible adoption, and passing pytest suites.
- Reported end-to-end performance remains limited in this setting, with the best overall pass rate at 40.2% for Claude-Sonnet-4.5 using Claude Code, while GPT-5 is reported at 21.7%.
- The paper’s own results suggest that execution dynamics and task visibility materially affect outcomes: GPT-5 and Qwen3-Thinking show high non-finish or early-stop rates, task_tracker usage is strongly correlated with performance, and revealing all test cases substantially improves Claude-Sonnet-4.5 results.
