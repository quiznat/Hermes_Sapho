- The comparison tested four tools: Claude Code, Cursor using Claude Sonnet 4, Gemini CLI, and OpenAI Codex.
- The source says Cursor uses a local-filesystem RAG-like system to gather codebase context, which could give it an advantage.
- The source states Gemini CLI used the Pro 2.5 model and had a 1M-token context window, the largest among the tested models.
- The benchmark used two test sets: a blank-repo Next.js app generation task and production tasks on a large Go API monorepo plus an Astro.js website.
- The production-code comparison gave different tasks to different tools, and the author says scoring there was based primarily on overall impression rather than identical task execution.
- In the vibe-coding test, Cursor received 9/10 and required 3 follow-up error prompts.
- Cursor was the only tested agent that successfully created both a Docker Compose file with a database and a SQL migration script for the required table in the vibe-coding task.
- In the vibe-coding test, Claude Code received 7/10, required 4 follow-up error prompts, and the deployed functionality worked on the first try.
- In the vibe-coding test, OpenAI Codex received 5/10, required 4 follow-up error prompts, and missed both Docker Compose and SQL migration scripts.
- In the vibe-coding test, Gemini CLI received 3/10, required 7 follow-up error prompts, and did not display error or success messages in the app.
- On the backend production task, Claude Code could not initially configure the go-quartz library and then switched to web search and GitHub documentation.
- On the backend production task, Cursor completed the refactor on the first try while matching existing dependency-injection and interface-returning patterns, but needed minor clarification about the Kubernetes namespace source.
- On the backend production task, Gemini CLI spent time loading broad codebase context and then completed the refactor on the first try with error handling.
- On the backend production task, Codex’s connection-retry failsafe worked correctly on the first try, with one follow-up prompt used to align standards and formatting.
- The final benchmark table gives Cursor scores of Setup 9, Cost 5, Quality 9, Context 8, Integration 8, Speed 9, Specialized 8, with an average of 8.
- The final benchmark table gives Claude Code scores of Setup 8, Cost 6, Quality 7, Context 5, Integration 9, Speed 7, Specialized 6, with an average of 6.8.
- The final benchmark table gives Gemini CLI scores of Setup 6, Cost 8, Quality 7, Context 9, Integration 5, Speed 5, Specialized 8, with an average of 6.8.
- The final benchmark table gives OpenAI Codex scores of Setup 3, Cost 6, Quality 8, Context 7, Integration 4, Speed 7, Specialized 7, with an average of 6.
